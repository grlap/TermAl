08Power with visible boundaries

Workflow controls are not a magic sandbox

Know what runs.
Know where.

TermAl is a powerful local development tool. It can launch agents, execute commands, edit files, and route work to remote machines when you allow it.

01⌂

Local by default

The browser and Rust backend bind to loopback. Sessions and settings are stored under ~/.termal/.

02◎

Agent rules still apply

Your selected CLI and model provider process data under their own authentication, sandbox, network, and data policies.

03!

Keep the API private

The local API can run commands and change files. Do not expose port 8787 directly to an untrusted network.

Good questions before you start

Is TermAl an AI model?+

No. TermAl is the operating environment around supported coding-agent CLIs. You install and authenticate those agents separately.

Does TermAl replace my editor?+

No. It includes files, Git, diffs, source panes, and a workspace terminal, but it is designed around supervising agent work—not replacing every editor or debugger workflow.

Are approvals a security sandbox?+

No. Approvals are visible workflow controls. The effective security boundary also depends on the selected agent, its sandbox and approval settings, the operating system, and any remote machine involved.

Can two people edit the same workspace together?+

Not as a collaborative multi-user editor today. Multiple browser layouts are useful across screens, but simultaneous writes to the same saved layout are last-write-wins.
